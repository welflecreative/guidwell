User Guide

Everything about using Guidwell

From embedding the wizard on your site to building branching logic trees — every feature covered tab by tab, with annotated mockups.

What is Guidwell?

Guidwell is a WordPress plugin that embeds a guided, step-by-step wizard on any page. Visitors answer a short series of questions and receive a personalized plan recommendation. Everything runs inside WordPress — your data stays in your database, not a third-party SaaS platform.

At its core, Guidwell works by assigning weights to each answer. When a visitor completes the wizard, those weights are tallied per plan and the highest-scoring plan is recommended. You control the questions, answers, plans, and scores — no code required.

ℹ️
Guidwell supports two modes: Linear (every visitor answers the same questions in order) and Tree (each answer can branch to a different next question). Linear mode works on all plans; branching logic requires Starter or higher.

Embedding the Wizard

Once the plugin is activated and your wizard is saved, place it on any page using a shortcode or the Gutenberg block.

Shortcode

[guidwell]

Uses the first published wizard automatically. To embed a specific wizard by its post ID:

[guidwell id="42"]

Gutenberg Block

In the block editor, search for Guidwell in the block inserter. The block renders a live preview in the editor and includes a sidebar control to select which wizard to embed.

✅
The wizard loads asynchronously — the page loads fast, then the wizard initializes. Safe to place anywhere on a page, including inside page builders and sidebar widgets.

Questions

Each question has a Question text field and a list of Answers. For each answer, you assign a weight (0–10) for every plan.

  • Reorder questions — drag the ≡ handle on the left of a question row
  • Delete a question — click the × on the right of the sidebar item
  • Add an answer — click + Add Answer inside the question editor
  • Reorder answers — drag them by their handles within the question editor
🔒
Free plan: limited to 5 questions per wizard. Upgrade to Starter or higher to remove this limit.

Plans

Plans are the recommendations your wizard matches visitors to. Each plan has:

  • Name — shown on the result screen (e.g. "Pro Retainer")
  • Price — displayed prominently (e.g. "$1,000/month")
  • Description — one or two sentences about what's included
  • CTA Label & URL — the button text and link on the result card
  • Tier — read-only, controls tie-breaking (lower = more accessible)
  • Features — checkboxes from your Feature Library (see Features tab)

You need at least 2 plans. Click + Add Plan at the bottom of the Plans section to add more. The delete button is hidden when only 2 plans remain.

Logic (Conditional Branching)

The Logic tab is a visual canvas builder for branching wizard paths. Instead of every visitor following the same sequence, each answer can send the visitor to a different next question.

🔒
Starter+ required for branching logic. The Logic tab is visible on all plans but branching is only saved and enforced at Starter and above.

Drawing a branch

  1. Click any answer row on a card. The row highlights and your cursor turns into a crosshair.
  2. Move your cursor toward another card. When you hover over a card header, it highlights as a drop target.
  3. Click the target card's header to complete the connection. A bezier wire is drawn from the answer to the destination.

To remove a connection, click the answer row that already has a wire — the connection is cleared.

Chaining questions

Drag a card close to the right edge of another card. When their headers align vertically, the card snaps in as a default next chain. The dashed wire between chained cards means "go here when no explicit branch applies." To break a chain, drag the card away.

Linear vs. Tree mode

Drawing your first branch automatically switches the wizard to tree mode. To switch back to linear, click Switch back to linear in the top-right of the Logic tab. This clears all branch connections.

Feature Library

The Features tab is a shared library of feature labels (e.g. "Monthly strategy call") that can be assigned to any plan. Assigned features appear as a checklist on the result screen.

Adding features

Type a label in the input and press Enter or click Add Feature. Feature names must be unique. Maximum 100 features.

Editing & deleting

Each row has an Edit button (inline text edit, confirm with Enter or ✓) and a × delete button. Deleting removes the feature from all plans that had it assigned.

Assigning to a plan

Go to the Wizard Builder tab, click a plan, and scroll to the Features section. Check any features to include for that plan. Changes are saved when you click Save.

ℹ️
The Features tab auto-saves immediately when you add, edit, or delete a feature — no need to click the main Save button for feature library changes.

Settings

The Settings tab has two sections: Colors & Branding and Import / Export.

Colors & Branding Starter+

Four color values control the wizard's appearance on the frontend:

  • Primary Color — buttons, selected answer states, progress bar
  • Primary Dark — hover states
  • Background — the panel behind the wizard card
  • Card Background — the wizard card itself

Export saves your entire wizard configuration as a .json file — useful for backups or copying a wizard to another site.

Import loads a previously exported JSON file and replaces the current wizard configuration in the builder. Review the result and click Save to persist.

⚠️
Importing overwrites the current unsaved state in the builder. Save first if you have unsaved edits.

Notifications

The Notifications tab controls two email types: admin notifications (you receive an email when someone completes the wizard) and visitor email capture (the wizard collects the visitor's email and optionally sends them their results).

🔒
Starter+ required for email capture and admin notifications.

Admin Result Notifications

When Send notification on result is enabled, you receive an email each time a visitor completes the wizard. Configure: recipient email & name, sender name & email, subject line, and optional header/footer text.

Visitor Email Capture

When Collect visitor email is enabled, a name and email field appears on the result screen before the visitor sees their recommendation. The collected contact info is included in your admin notification email.

Custom SMTP

By default, Guidwell uses WordPress's wp_mail(). For unreliable hosting mail, configure a dedicated SMTP server with host, port, username, password, and encryption (TLS / SSL / none). Click Send Test Email to verify the connection before saving.

✅
The SMTP password is stored encrypted (AES-256-CBC) and never exposed in the admin UI after it's been set. The field shows a placeholder when a password is already saved.

Scoring & Weights

Guidwell uses a weighted scoring matrix. Every answer has a score (0–10) for every plan. When a visitor picks an answer, those scores are added to a running total per plan. After all questions are answered, the plan with the highest total is recommended.

Setting weights

In the Wizard Builder, click any question to open it. Each answer row shows a number input (0–10) for every plan. Guidelines:

  • 0 — this answer has no signal for this plan
  • 3–5 — moderate signal
  • 10 — strongest possible signal toward this plan

Weights don't need to add up to anything specific — they're relative scores. Typically each answer should have its highest weight on one plan and lower or zero weights on the others.

Tie-breaking

If two plans tie on score, the plan with the lower tier number wins. Tier 1 is your most accessible option. Ties always favor the lower-cost option, which tends to feel more honest to visitors.

ℹ️
The result screen shows the top 3 matching plans (not just the winner), each with a score bar. This helps visitors self-select into higher tiers when they see a near-tie.

What visitors see

  • The top recommended plan with name, price, description, and CTA button
  • An insight paragraph explaining why this plan fits their answers
  • A score comparison for the top 3 matching plans
  • A feature checklist for the recommended plan (if features are assigned)
  • A Restart button to go back through the wizard

If visitor email capture is enabled, a name and email form appears at the top of the modal before the result is revealed.

Saving

The Save button lives in the top-right of the tab bar and is disabled until you make a change.

  • Saving is global — one click saves changes across questions, answers, plans, and logic connections.
  • The Features tab auto-saves immediately — feature changes don't wait for Save.
  • The Settings and Notifications tabs have their own individual Save buttons.
  • If a save fails, the button shows an error state and a notice banner appears below the tab bar.
⚠️
Navigating away without saving discards unsaved changes. There is no auto-save for the main wizard config.

Shortcode Reference

Basic usage

[guidwell]

Embeds the first published wizard. No parameters required for a single-wizard setup.

Specific wizard by ID

[guidwell id="42"]

Embeds a specific wizard by its WordPress post ID. Find the ID by hovering over the wizard title in the post list — the ID appears in the URL (e.g. post=42).

Gutenberg block

Search for Guidwell in the block inserter (+ button). The block includes a wizard selector dropdown — no ID needed.

ℹ️
The wizard loads asynchronously and is safe to embed inside any block or page builder. Each page load is a fresh visitor session — refreshing the page resets the wizard to the first question.
