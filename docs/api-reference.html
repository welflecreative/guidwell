Developer Reference

API Reference

REST API endpoints, PHP constants, the tier system, and the GitHub release pipeline — everything a developer needs to integrate with or extend Guidwell.

REST API Overview

Guidwell registers its own REST API namespace at guidwell/v1. All endpoints are available at:

https://yoursite.com/wp-json/guidwell/v1/...

Write endpoints require WordPress authentication. The admin React app sends a X-WP-Nonce header using a nonce generated via wp_create_nonce('wp_rest') and localized on the admin page. Public endpoints (/config/{id}, /features, /send-result) do not require authentication.

Authentication levels

  • Admin — requires manage_options capability (WordPress Administrators only)
  • Public — no authentication required; public endpoints are rate-limited where applicable

Create Wizard

POST /guidwell/v1/wizard Admin

Creates a new guidwell_wizard post. Enforces the active tier's active_wizards limit before creating.

ParameterTypeRequiredDescription
titlestringoptionalWizard post title. Defaults to "My Wizard".

Response: { "id": 42 } — the new wizard's post ID.

Get Wizard Config

GET /guidwell/v1/config/{wizard_id} Public

Returns the full JSON configuration for a published wizard. Used by the frontend wizard component to load questions, plans, and logic tree connections. Returns 400 if the wizard has no config yet, 404 if the wizard doesn't exist or isn't published.

ParameterTypeRequiredDescription
wizard_idintegerrequiredWordPress post ID of the wizard. Must be ≥ 1.

Response: The full wizard config object (questions array, plans array, mode string).

ℹ️
The wizard config is also served via wp_localize_script on pages that contain the shortcode or block, so the frontend doesn't need to make a separate API call on initial load. The REST endpoint is used for subsequent loads and admin preview.

Save Wizard Config

POST /guidwell/v1/config/{wizard_id} Admin

Validates, sanitizes, and saves the wizard configuration. Enforces tier limits (question count, tree depth, tree nodes). Returns the sanitized config on success.

Body: JSON object with the following top-level keys:

KeyTypeDescription
questionsarrayArray of question objects. Each must have id, text, and answers array.
plansarrayArray of plan objects. Minimum 2. Each must have slug, tier, name, ctaLabel.
modestring"linear" or "tree". Defaults to "linear" if omitted or invalid.
_importedbooleanSet to true to flag as a JSON import. Requires json_import tier feature; stripped before saving.

Question object:

{
  "id": "q1",          // sanitize_key()
  "text": "...",       // sanitize_text_field()
  "answers": [
    {
      "id": "a1",       // sanitize_key()
      "label": "...",  // sanitize_text_field()
      "weights": { "plan-slug": 3 },
      "next": null    // or "q2" for tree mode
    }
  ]
}

Plugin Settings

GET /guidwell/v1/settings Admin

Returns the current plugin display settings (colors and branding options).

POST /guidwell/v1/settings Admin

Saves display settings. All color values are validated with sanitize_hex_color().

KeyTypeDescription
primaryColorstringHex color. Button and accent color.
primaryDarkstringHex color. Hover states.
backgroundColorstringHex color. Wizard panel background.
cardBackgroundstringHex color. Wizard card background.
useThemeColorsbooleanAuto-detect colors from active theme.

Contact & SMTP Settings

GET /guidwell/v1/contact-settings Admin

Returns contact and SMTP settings. The smtpPassword field is always returned as an empty string — use smtpPasswordSet: true/false to check if a password is stored.

POST /guidwell/v1/contact-settings Admin

Saves contact and SMTP settings. SMTP password is AES-256-CBC encrypted before storage using a key derived from AUTH_KEY. Omit smtpPassword to keep the existing stored password.

KeyTypeDescription
recipientEmailstringAdmin notification recipient.
recipientNamestringRecipient display name.
senderEmailstringFrom address. Defaults to WordPress admin email.
senderNamestringFrom display name.
emailSubjectstringNotification email subject line.
sendOnResultbooleanEnable admin notifications.
collectVisitorEmailbooleanShow email capture on result screen.
useCustomSmtpbooleanUse custom SMTP instead of wp_mail().
smtpHoststringSMTP server hostname.
smtpPortintegerSMTP port (1–65535).
smtpUsernamestringSMTP authentication username.
smtpPasswordstringSMTP password (plain text — encrypted before storage).
smtpEncryptionstring"tls", "ssl", or "none".

Send Result Email

POST /guidwell/v1/send-result Public · Rate limited

Sends the result notification email after a visitor completes the wizard. Rate-limited to 5 requests per IP per hour. No-ops if sendOnResult is disabled in contact settings.

KeyTypeDescription
wizardIdintegerThe wizard's post ID.
recommendedPlanobjectrequired Must have a non-empty name field.
runnerUpPlanobject|nullSecond-best plan, optional.
insightstringThe generated insight paragraph.
visitorEmailstringVisitor's email for the visitor copy.

Feature Library

GET /guidwell/v1/features Public

Returns the feature library as a JSON array of { id, label } objects. Used by the frontend to render feature checklists on the result screen.

POST /guidwell/v1/features Admin

Replaces the entire feature library. Maximum 100 features. Each item requires a non-empty id (sanitized to slug) and label (max 120 chars).

PHP Constants

Define these in wp-config.php before the "stop editing" line to customize Guidwell's behavior at the server level.

ConstantTypeDescription
GUIDWELL_INTERNAL_FLAG boolean Set to true to bypass all tier enforcement and grant Agency-level access. Intended for the plugin owner's install only. Takes precedence over the guidwell_license_tier database option.
GUIDWELL_VERSION string Set automatically from the plugin header. Do not override.
GUIDWELL_PLUGIN_DIR string Absolute path to the plugin directory (trailing slash). Set automatically.
GUIDWELL_PLUGIN_URL string Full URL to the plugin directory (trailing slash). Set automatically.

Internal configuration example

// wp-config.php — above "That's all, stop editing!"
define( 'GUIDWELL_INTERNAL_FLAG', true );

Guidwell_Tiers PHP Class

The Guidwell_Tiers class is the single access point for all tier and feature gating logic. Available after the plugin is loaded.

Methods

// Get the active tier slug for this install
Guidwell_Tiers::current(): string
// Returns: 'free' | 'starter' | 'pro' | 'agency'

// Check if the current tier includes a feature
Guidwell_Tiers::can( string $feature ): bool
// Feature keys: 'custom_branding', 'email_capture', 'result_notifications',
//   'json_export', 'json_import', 'conditional_logic', 'analytics_basic',
//   'analytics_full', 'white_label', 'email_integrations',
//   'unlimited_wizards', 'multi_site'

// Get a numeric limit for the current tier (null = unlimited)
Guidwell_Tiers::limit( string $key ): ?int
// Limit keys: 'questions_per_wizard', 'active_wizards',
//   'sites', 'logic_tree_depth', 'logic_tree_nodes'

// True if $count exceeds the tier's limit for $key
Guidwell_Tiers::exceeds_limit( string $key, int $count ): bool

// Returns full tier summary (for JS / REST responses)
Guidwell_Tiers::current_summary(): array

Setting the tier (database)

When no GUIDWELL_INTERNAL_FLAG constant is present, the active tier is read from the WordPress options table:

// Via WP-CLI
wp option set guidwell_license_tier agency

// Valid values: 'free', 'starter', 'pro', 'agency'

Release Pipeline

Guidwell uses two GitHub Actions workflows for versioning and distribution.

Bump Version (.github/workflows/bump-version.yml)

Triggered manually from GitHub → Actions → Bump Version → Run workflow. Takes a bump input (patch, minor, or major).

Reads the current version from guidwell.php, increments the correct segment, updates both the plugin header comment and the GUIDWELL_VERSION constant, commits the change to main, and pushes a vX.Y.Z tag.

Release (.github/workflows/release.yml)

Triggered automatically on any v* tag push. Runs:

  1. npm ci && npm run build — compiles fresh JS bundles
  2. composer install --no-dev — installs runtime PHP dependencies only
  3. Rsync with .distignore exclusions into a clean temp directory
  4. Zips into guidwell-{version}.zip with the required single root folder
  5. Creates a GitHub Release and attaches the zip as a release asset

Auto-updates on installed sites

Installed copies of the plugin include the Plugin Update Checker library wired to GitHub Releases. When a new release is published, WordPress will show the standard update prompt in WP Admin → Plugins within 12 hours (or on the next admin page load).

✅
The update checker requires the GitHub repository to be public to fetch release metadata without authentication. If the repo is private, a GitHub token must be provided in the plugin's update checker configuration.
