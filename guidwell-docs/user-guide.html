User Guide

Everything about using Guidwell

From embedding the wizard on your site to building branching logic trees — every feature covered tab by tab, with annotated mockups.

What is Guidwell?

Guidwell is a WordPress plugin that embeds a guided, step-by-step wizard on any page. Visitors answer a short series of questions and receive a personalized plan recommendation. Everything runs inside WordPress — your data stays in your database, not a third-party SaaS platform.

At its core, Guidwell works by assigning weights to each answer. When a visitor completes the wizard, those weights are tallied per plan and the highest-scoring plan is recommended. You control the questions, answers, plans, and scores — no code required.

ℹ️
Guidwell supports two modes: Linear (every visitor answers the same questions in order) and Tree (each answer can branch to a different next question). Linear mode works on all plans; branching logic requires Starter or higher.

Embedding the Wizard

Once the plugin is activated and your wizard is saved, place it on any page using a shortcode or the Gutenberg block.

Shortcode

[guidwell]

Uses the first published wizard automatically. To embed a specific wizard by its post ID:

[guidwell id="42"]

Gutenberg Block

In the block editor, search for Guidwell in the block inserter. The block renders a live preview in the editor and includes a sidebar control to select which wizard to embed.

✅
The wizard loads asynchronously — the page loads fast, then the wizard initializes. Safe to place anywhere on a page, including inside page builders and sidebar widgets.

Questions

Each question has a Question text field and a list of Answers. For each answer, you assign a weight (0–10) for every plan using the small number inputs below the answer text.

  • Reorder questions — drag the ☰ handle on the left
  • Delete a question — hover the sidebar item and click ×
  • Add an answer — click + Add Answer in the right panel
  • Reorder answers — drag them by their ☰ handles
🔒
Free plan: limited to 5 questions per wizard. The sidebar shows a tier-gate notice when you hit the limit. Upgrade to Starter or higher to remove it.

Plans

Plans are the recommendations your wizard matches visitors to. Click any plan in the sidebar to edit it. Each plan has: Name, Price, Description, CTA Label & URL, and a Features checklist drawn from your Feature Library.

The delete button on a plan is hidden when only 2 plans remain — that's the minimum.

Drawing a branch

  1. Click any answer row on a card. The row highlights teal and your cursor changes to a crosshair.
  2. Move toward another card. When you hover over its header, it turns green as a valid drop target.
  3. Click the target header to complete the wire. The answer row now shows the destination name.

To remove a wire, hover the answer row that has one and click the × that appears on the right side of the row.

Chaining (default next)

Drag a card close to the right edge of another card until the edges snap flush. This sets a default next — if a visitor picks an answer with no explicit wire, they advance to the chained card. The chained card shows a squared-off left edge and a footer that reads "→ chains to: [name]".

Feature Library

The Features tab manages a shared list of feature labels (e.g. "Monthly strategy call") that can be checked on each plan. Checked features appear as a bullet list on the result screen.

The Features tab auto-saves immediately — adding, editing, or deleting a feature takes effect right away without clicking the main Save button.

To assign features to a plan, go to the Wizard Builder tab, click a plan, and check features in the Features section of the plan editor.

Custom SMTP

Enable Use custom SMTP to configure host, port, username, password, and encryption (TLS / SSL / none). The password is stored encrypted (AES-256-CBC) and never re-exposed in the UI. Click Send Test Email to verify the connection before saving.

Scoring & Weights

Every answer has a numeric weight for each plan. When a visitor picks an answer, those values are added to a running total per plan. After all questions, the plan with the highest total is recommended.

Setting weights

In the Wizard Builder, select a question. Under each answer you'll see a row of small number inputs — one per plan, labeled with the plan name. The maximum weight is equal to the number of plans you have configured — so with 3 plans the range is 0–3, with 4 plans it's 0–4, and so on. The input enforces this automatically. Guidelines:

  • 0 — no signal for this plan from this answer
  • 1 — weak signal
  • Max (= plan count) — strongest possible signal toward this plan

Weights are relative — a plan with total 9 beats one with 7. Typically each answer should have its highest weight on one plan and lower weights on the others.

Tie-breaking

If two plans tie, the plan with the lower tier number wins. Tier 1 is your most accessible option — ties always favour the lower-cost plan, which feels honest to visitors.

ℹ️
The result screen shows the top 3 matching plans, each with a score bar — not just the winner. Visitors can self-select upward when they see a near-tie.

Saving

The Save button sits in the tab bar and is dimmed (disabled) until you make a change. One click saves all edits across questions, answers, plans, and logic connections.

  • Features tab — auto-saves immediately, no Save button needed
  • Settings tab — has its own Save Settings button
  • Notifications tab — has its own Save Notifications button
  • Navigating away without saving discards unsaved changes — there is no auto-save for the builder

Shortcode Reference

Basic usage

[guidwell]

Embeds the first published wizard. No parameters required for a single-wizard setup.

Specific wizard by ID

[guidwell id="42"]

Find the post ID by hovering over a wizard in the post list — the ID appears in the URL bar (post=42).

Gutenberg block

Search for Guidwell in the block inserter (/guidwell or via the + button). The block includes a wizard-selector dropdown and renders a live preview in the editor.
