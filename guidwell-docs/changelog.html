Release History

Changelog

All notable changes, fixes, and security patches. Follows Semantic Versioning — patch for fixes, minor for new features, major for breaking changes.

v1.4.2
Latest Feature 2026-06-28
Added
  • Multi-select questions: Question editor now supports an "Allow multiple selections" mode — visitors can pick more than one answer before advancing. All selected answers contribute their weights to scoring.
  • Zero-score plan filter: Plans with a total weighted score of 0 are now excluded from results. Visitors only see plans that genuinely match their answers.
  • Docs auto-deploy: The release workflow now syncs the guidwell-docs/ folder to the live documentation server on every release — no manual uploads required.
Fixed
  • Watermark upgrade UX: The "Remove watermark" setting now shows as a disabled checkbox with an inline upgrade pill for lower-tier users, instead of replacing the control with a lock badge. Makes the feature discoverable rather than hidden.
  • removeWatermark persistence: The setting was not being saved to the database due to a missing entry in the API allowlist. Fixed — the server-side tier gate is now enforced correctly.
v1.4.0
Fix 2026-06-22
Fixed
  • Gutenberg full-bleed alignment: The #guidwell container now renders correctly inside Gutenberg's constrained layout. Added alignfull class + JS viewport correction to handle the margin-inline: auto !important override applied by Gutenberg's .is-layout-constrained rule.
  • Mobile card overflow: Changed card minimum width from clamp(600px, ...) to min(clamp(360px, ...), 100%) so the wizard card never overflows narrow viewports (e.g. Pixel 7 at 412px).
  • Logo SVG crop: Trimmed excess whitespace from the viewBox of the Guidwell logo SVG so it renders crisply without phantom padding.
  • Logo cache-busting: SVG logo URL now includes ?ver=x.y.z so browsers pick up the corrected logo on update.
  • Mount-point DRY: Extracted guidwell_mount_point() helper to eliminate identical HTML string duplication across the shortcode, Gutenberg block, and Elementor widget.
v1.1.0
Feature 2026-06-17
Added
  • Logic tree canvas: Visual branching editor on a dot-grid canvas. Click answer rows to draw bezier wires to destination questions. Horizontal chain connections for default-next flow. Depth and node limits enforced per tier.
  • Linear / Tree mode toggle: Wizards can run in linear mode (all visitors same path) or tree mode (each answer can branch to a different next question).
  • Multi-site licensing: Pro licenses now activate on up to 5 WordPress installs; Agency on up to 25.
  • JSON import (Pro+): Import a previously exported wizard config to restore or duplicate a wizard across sites.
  • Plan icons (Pro+): Upload a logo or icon image to each plan for a richer results card.
  • Basic analytics (Pro+): See which plans are recommended most often.
v1.0.1
Security 2026-06-16
Security — High
  • Capability escalation fix: can_edit() in the REST API now requires manage_options instead of edit_posts. Previously, any WordPress author or editor could create wizards and overwrite existing wizard configurations via the REST API.
  • Admin menu gating: The Guidwell admin menu now requires manage_options (was edit_posts). Non-admin users no longer see the menu or receive a wp_rest nonce from the admin page.
  • Wizard config sanitization: All string fields in wizard configs (question text, answer labels, plan names, descriptions, CTA labels, slugs, URLs) are now sanitized through sanitize_text_field(), sanitize_key(), and esc_url_raw() before storage. Closes a latent stored XSS vector.
Security — Medium
  • Email header injection fix: CR/LF characters are now stripped from recipient and sender name fields before being composed into mail headers. Prevents header injection attacks via crafted name values containing newline sequences.
  • SMTP encryption none honored: The SMTP configuration now correctly sets SMTPSecure = '' and disables SMTPAutoTLS when encryption is set to none, instead of silently falling back to TLS.
Security — Low
  • Missing tiers.json detection: Guidwell_Tiers::config() now calls wp_die() with a clear reinstall prompt if tiers.json cannot be read. Previously, a missing file produced an empty config that silently granted unlimited access to all limits.
Added
  • GitHub release pipeline: .github/workflows/release.yml — pushing a v* tag builds the plugin zip and creates a GitHub Release automatically.
  • Version bumping workflow: .github/workflows/bump-version.yml — trigger from GitHub Actions UI to bump patch, minor, or major version, commit, and tag in one step.
  • Auto-update support: Installed copies now detect new GitHub Releases and surface the standard WordPress update prompt in WP Admin.
v1.0.0
Initial Release 2026-06-10
Added
  • Wizard Builder: Drag-and-drop question and answer editor with per-answer per-plan weight assignment. Minimum 1 question, 2 plans.
  • Weighted scoring engine: Sums answer weights per plan, recommends the highest-scoring plan, breaks ties by tier number. Returns top 3 matches with score bars.
  • Logic tree canvas: Visual branching editor on a dot-grid canvas. Click answer rows to draw bezier wires to destination questions. Horizontal chain connections for default-next flow.
  • Result modal: Full-screen overlay with recommended plan, insight paragraph, feature checklist, CTA button, and score comparison. Closeable via ×, Escape, or overlay click.
  • Feature Library tab: Shared feature label library assignable to any plan. Auto-saves immediately.
  • Settings tab: Custom colors (primary, primary dark, background, card), use-theme-colors toggle, JSON export/import.
  • Notifications tab: Admin result notifications, visitor email capture, custom SMTP with AES-256-CBC password encryption.
  • Tier system: Free, Starter ($79/yr), Pro ($149/yr), Agency ($299/yr) — enforced via Guidwell_Tiers class reading from config/tiers.json.
  • Shortcode embed: [guidwell] and [guidwell id="42"].
  • Gutenberg block: Native block with wizard selector and live editor preview.
  • REST API: Full guidwell/v1 namespace with endpoints for wizard CRUD, config, settings, contact settings, features, and result email delivery.
  • Rate limiting: Public /send-result endpoint limited to 5 requests/IP/hour via WordPress transients.
