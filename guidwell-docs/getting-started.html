Getting Started

Install & set up Guidwell

From zero to a live wizard on your WordPress site — download the plugin, upload it, configure your first questions and plans, and embed it on a page.

Requirements

Guidwell runs entirely inside WordPress. No external services or API keys are required for core functionality.

RequirementMinimum
WordPress6.0 or higher
PHP8.1 or higher
MySQL / MariaDBWordPress minimum (5.5+ / 10.1+)
Browser (admin)Any modern browser
Server outgoing mailRequired for email notifications (optional)
ℹ️
PHP 8.1+ is required. PHP 7.x is not supported. Most managed WordPress hosts (WP Engine, Kinsta, Flywheel, SiteGround) default to PHP 8.1 or 8.2. Check your hosting control panel if unsure.

Installation

Guidwell is distributed as a .zip file via GitHub Releases. Download the latest release and upload it through the WordPress admin.

Step-by-step

  1. Go to github.com/welflecreative/guidwell/releases and download the latest guidwell-x.y.z.zip asset from the release.
  2. In your WordPress admin, navigate to Plugins → Add New → Upload Plugin.
  3. Click Choose File, select the guidwell-x.y.z.zip you downloaded, and click Install Now.
  4. Once the upload completes, click Activate Plugin.
  5. A new Guidwell menu item appears in the left sidebar of WP Admin. Click it to open the plugin.
✅
Future updates are automatic. Once the plugin is installed from the release zip, WordPress will detect new releases on GitHub and show an update prompt in WP Admin → Plugins — just like any other plugin.

Manual installation (FTP)

If you prefer FTP or your host blocks file uploads:

  1. Extract the guidwell-x.y.z.zip file. You should get a single guidwell/ folder.
  2. Upload the entire guidwell/ folder to wp-content/plugins/ on your server.
  3. In WP Admin, go to Plugins → Installed Plugins, find Guidwell in the list, and click Activate.

Activation & License

The Free tier requires no license key — activate the plugin and it works immediately with free-tier limits (5 questions per wizard, 1 active wizard, 1 site).

For paid tiers (Starter, Pro, Agency), license activation will be handled through the Guidwell settings once the licensing layer is live. In the meantime, your license tier is set directly in the WordPress options table after purchase — Welfle Creative will provide instructions with your purchase confirmation.

ℹ️
The license tier controls which features and limits are active. You can check your current tier at any time in WP Admin → Guidwell — the tier name is shown in the top-right of the admin interface.

Your First Wizard

When you open WP Admin → Guidwell for the first time, a starter template is pre-loaded with sample questions and plans. A blue banner confirms this — your wizard exists only in memory at this point and hasn't been saved yet.

  1. Click any question in the left sidebar to open it. Review the question text and the answers.
  2. Edit the question text, answer labels, and weights to match your service. Each answer has a weight (0–10) per plan — higher weight = stronger signal toward that plan.
  3. Click a plan name in the lower sidebar section to edit it. Fill in the plan name, price, description, and CTA button URL.
  4. When you're ready, click Save in the top-right of the tab bar. The wizard is created in the database and the Save button returns to its disabled (greyed-out) state.
⚠️
Save before navigating away. There is no auto-save for the main wizard config. Closing the tab or navigating away without clicking Save discards your changes.

Minimum requirements to save

  • At least 1 question with at least 2 answers
  • At least 2 plans, each with a name and CTA label
  • Each answer must have a weight assigned for each plan (0 is valid)

Embedding the Wizard

Once your wizard is saved, place it on any WordPress page or post using a shortcode or the Gutenberg block.

Shortcode (simplest)

Add the following to any page or post:

[guidwell]

This automatically uses the first published wizard. To embed a specific wizard by its WordPress post ID:

[guidwell id="42"]

Gutenberg block

In the block editor, click the + block inserter and search for Guidwell. Add the block to your page. The block renders a live preview of the wizard in the editor and includes a sidebar control to select which wizard to embed.

Where to find your wizard's post ID

The wizard's ID is visible in the URL when you look at it in the WordPress post list. Navigate to WP Admin → Posts → All Posts, filter by post type if needed, and hover over the wizard title — the ID appears in the URL in the status bar (e.g. post=42).

✅
The wizard loads asynchronously — the page itself loads fast, then the wizard initializes. It's safe to place the shortcode anywhere on a page, including inside page builders, inside other blocks, or in a sidebar widget.

Testing It Live

After embedding the shortcode, visit the page on the frontend as a regular visitor would.

What to verify

  • The wizard card appears and shows your first question
  • Clicking an answer selects it (highlighted state) and the Next button becomes active
  • Progressing through all questions leads to the result modal
  • The result modal shows the correct recommended plan based on your weights
  • The CTA button links to the correct URL
  • The modal closes with the × button, Escape key, or clicking the overlay

Checking which plan gets recommended

Pick a specific path through your wizard (choose the same answers each time) and note which plan wins. Cross-check manually: sum the weights for each chosen answer per plan and verify the highest total matches the recommendation. If results seem off, review the weight values in the Wizard Builder.

Testing notifications

If you have admin result notifications enabled in the Notifications tab, complete the wizard with a visitor email address and verify you receive the notification email. Use the Send Test Email button in the Notifications tab to verify SMTP connectivity before testing the full flow.

ℹ️
The wizard uses your browser's local state — refreshing the page resets it back to the first question. This is intentional: each page load is a fresh visitor session.
